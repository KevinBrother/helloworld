# UI 开发规则

当进行界面设计与实现时，必须遵循以下规则：

## 设计一致性

- 项目使用 **shadcn-ui** 作为基础组件库
- **所有新设计必须保持与现有设计风格一致**
- 新增组件应遵循 `packages/ui-components` 中现有组件的设计模式
- 参考 `docs/STYLE_GUIDE.md` 了解样式系统规范

## Storybook 要求

当在 `packages/ui-components` 中新增或修改组件时：

- **新增组件**：必须创建对应的 Storybook 文档
- **修改组件**：检查是否有对应的 Storybook
  - 如果有 → 验证现有案例参数是否覆盖新的修改
  - 如果没有 → 必须添加 Storybook 文档

## 组件使用优先级

实现功能时按以下顺序寻找组件：

1. **优先使用现有组件**：从 `packages/ui-components` 中查找可复用的通用组件
2. **组合现有组件**：通过组合多个现有组件实现需求
3. **创建新组件**：仅在以下情况下考虑创建新的通用组件：
   - 组件足够通用
   - 预计会在 3 个及以上地方使用
   - 不包含业务逻辑

## 通用组件决策流程

当需要创建新组件时：

1. 先询问用户："这个组件足够通用吗？会在多处使用吗？"
2. 如果用户确认 → 在 `packages/ui-components` 中创建，并添加 Storybook
3. 如果用户否定 → 在应用层代码中实现，不放入 ui-components

## ui-components 包原则

- **只包含**可复用的、无业务逻辑的组件
- **不包含**特定功能实现
- 组件必须是通用的，可跨多个功能使用
