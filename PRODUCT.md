# Product

## Register

product

## Users

Developers and automation builders who define, inspect, debug, and run contract-first RPA workflows. They work in a local development context, moving between AST files, block manifests, generated UI projections, diagnostics, and execution results.

## Product Purpose

RPA Agent Workflow defines, validates, compiles, executes, debugs, and visualizes workflow contracts. The web surface helps users inspect a projected workflow graph, select nodes, edit supported fields, review diagnostics, run the workflow through the local service when available, and export edit operations.

Success means the editor makes workflow structure and runtime state clear enough that users can safely understand what changed, why validation failed, and what the next executable action is.

## Brand Personality

Precise, technical, calm. The interface should feel like a dependable engineering tool: dense enough for real work, restrained in decoration, and explicit about local service state, diagnostics, and saved changes.

## Anti-references

Avoid marketing-page styling, decorative dashboard theatrics, heavy gradients, opaque glass panels, novelty controls, and visual treatments that make the workflow harder to scan. Do not make inactive states overly colorful or make the graph canvas compete with inspector details.

## Design Principles

1. Make workflow state visible: connection status, selected node, diagnostics, save state, and run output should be easy to locate without reading long copy.
2. Preserve engineering trust: use familiar controls, predictable panels, clear labels, and consistent status language.
3. Keep density organized: allow complex workflows and metadata without turning the screen into repeated cards or decorative panels.
4. Treat errors as first-class data: diagnostics should be actionable, attributable, and reachable from the task surface.
5. Prefer task speed over spectacle: motion and color should communicate state, not decorate.

## Accessibility & Inclusion

Target WCAG 2.2 AA for contrast, keyboard navigation, focus visibility, semantic controls, and reduced-motion support. The workflow editor should remain usable for keyboard users, low-vision users, and users who rely on clear text labels instead of color alone.
